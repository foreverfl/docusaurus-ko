# 다음은 무엇인가요?

축하합니다! 이제 Docusaurus의 대부분의 핵심 기능을 이해하셨습니다. 여러분은 다음을 수행했습니다:

- [페이지 플러그인을 사용하여](./creating-pages.mdx) 독립적인 React / Markdown 페이지를 만들었습니다
- [문서 플러그인을 사용하여](./docs/docs-introduction.mdx) 문서 페이지를 만들었습니다. 여기에는 [사이드바 구성](./docs/sidebar/index.mdx)과 심지어 [버전 관리](./docs/versioning.mdx)도 포함됩니다
- [블로그 플러그인을 사용하여](../blog.mdx) 완전한 기능을 갖춘 블로그를 만들었습니다
- 모든 콘텐츠 플러그인에 유용한 [다양한 Markdown 기능](./markdown-features/markdown-features-intro.mdx)을 직접 사용해 보았습니다
- [스타일시트](../styling-layout.mdx) 또는 [스위즐링](../swizzling.mdx)을 사용하여 사이트의 외관을 사용자 정의했습니다
- [이미지와 기타 자산을 페이지에 포함](../static-assets.mdx)시켰습니다
- [검색 기능을 사이트에 추가](../search.mdx)했습니다
- 표준 Docusaurus API를 통해 [브라우저 지원](../browser-support.mdx)과 [SEO](../seo.mdx)가 어떻게 이루어지는지 이해했습니다
- [개별 플러그인](../using-plugins.mdx)이 어떻게 설치되고 구성되는지 배웠습니다
- 사이트를 [콘텐츠 호스트에 배포](../deployment.mdx)했습니다
- 사이트를 [국제화](../i18n/i18n-tutorial.mdx)하여 여러 언어를 포함시켰습니다

이 시점에서 아마도 이미 큰 `docusaurus.config.js` 파일을 가지고 있을 것입니다😄 하지만 아직 많은 코드를 작성하지는 않았습니다! 대부분의 기능은 캡슐화된 Docusaurus API를 호출하여 구현되었습니다. 여러분의 여정을 계속하면서 세 가지 경로를 선택할 수 있습니다:

- 더 고급 Docusaurus 개념을 배웁니다. 이는 API가 수행하는 작업에 대해 더 깊이 이해하는 데 도움이 될 것입니다.
- [모든 기존 API](../docusaurus-core.mdx)에 대해 읽어봅니다. 가이드에서 다루지 않은 많은 API가 있습니다!
- [플러그인 개발](../api/plugin-methods/README.mdx)을 배워 사이트의 기능을 확장합니다.
