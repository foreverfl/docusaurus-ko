# 사이드바 항목

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import BrowserWindow from '@site/src/components/BrowserWindow';
```

이전 섹션의 예제에서 우리는 세 가지 항목 유형을 소개했습니다: `doc`, `category`, `link`. 이들의 사용법은 꽤 직관적입니다. 이제 공식적으로 이들의 API를 소개하겠습니다. 네 번째 유형인 `autogenerated`도 있는데, 이는 나중에 자세히 설명하겠습니다.

- **[Doc](#sidebar-item-doc)**: 문서 페이지로 연결하고, 사이드바와 연관짓습니다
- **[Link](#sidebar-item-link)**: 내부 또는 외부 페이지로 연결합니다
- **[Category](#sidebar-item-category)**: 사이드바 항목의 드롭다운을 생성합니다
- **[Autogenerated](autogenerated.mdx)**: 사이드바 슬라이스를 자동으로 생성합니다
- **[HTML](#sidebar-item-html)**: 항목 위치에 순수 HTML을 렌더링합니다
- **[\*Ref](multiple-sidebars.mdx#sidebar-item-ref)**: 문서 페이지로 연결하지만, 항목이 내비게이션 생성에 참여하지 않게 합니다

## Doc: 문서로 연결 {#sidebar-item-doc}

`doc` 유형을 사용하여 문서 페이지로 연결하고 해당 문서를 사이드바에 할당합니다:

```ts
type SidebarItemDoc =
  // 일반 구문
  | {
      type: 'doc';
      id: string;
      label: string; // 사이드바 라벨 텍스트
      className?: string; // 사이드바 라벨의 클래스 이름
      customProps?: Record<string, unknown>; // 사용자 정의 속성
    }

  // 축약 구문
  | string; // docId 단축키
```

예시:

```js title="sidebars.js"
export default {
  mySidebar: [
    // 일반 구문:
    // highlight-start
    {
      type: 'doc',
      id: 'doc1', // 문서 ID
      label: '시작하기', // 사이드바 라벨
    },
    // highlight-end

    // 축약 구문:
    // highlight-start
    'doc2', // 문서 ID
    // highlight-end
  ],
};
```

문서 축약형이나 [자동 생성](autogenerated.mdx) 사이드바를 사용하면 항목 정의를 통해 사이드바 라벨을 사용자 정의할 수 있는 기능을 잃게 됩니다. 그러나 해당 문서 내에서 `sidebar_label` Markdown 전면 사항을 사용할 수 있으며, 이는 사이드바 항목의 `label` 키보다 우선순위가 높습니다. 마찬가지로 `sidebar_custom_props`를 사용하여 문서 페이지에 대한 사용자 정의 메타데이터를 선언할 수 있습니다.

:::note

`doc` 항목은 [암시적 사이드바 연관](./multiple-sidebars.mdx#sidebar-association)을 설정합니다. 동일한 문서를 여러 사이드바에 할당하지 마세요: 대신 유형을 `ref`로 변경하세요.

:::

:::tip

사이드바 사용자 정의 속성은 임의의 문서 메타데이터를 클라이언트 측으로 전파하는 유용한 방법입니다. 따라서 문서 객체를 가져오는 문서 관련 훅을 사용할 때 추가 정보를 얻을 수 있습니다.

:::

## Link: 모든 페이지로 연결 {#sidebar-item-link}

`link` 유형을 사용하여 문서가 아닌 모든 페이지(내부 또는 외부)로 연결합니다.

```ts
type SidebarItemLink = {
  type: 'link';
  label: string;
  href: string;
  className?: string;
  description?: string;
};
```

예시:

```js title="sidebars.js"
export default {
  myLinksSidebar: [
    // highlight-start
    // 외부 링크
    {
      type: 'link',
      label: 'Facebook', // 링크 라벨
      href: 'https://facebook.com', // 외부 URL
    },
    // highlight-end

    // highlight-start
    // 내부 링크
    {
      type: 'link',
      label: '홈', // 링크 라벨
      href: '/', // 내부 경로
    },
    // highlight-end
  ],
};
```

## HTML: 사용자 정의 마크업 렌더링 {#sidebar-item-html}

`html` 유형을 사용하여 항목의 `<li>` 태그 내에 사용자 정의 HTML을 렌더링합니다.

이는 구분선, 섹션 제목, 광고, 이미지와 같은 사용자 정의 항목을 삽입하는 데 유용할 수 있습니다.

```ts
type SidebarItemHtml = {
  type: 'html';
  value: string;
  defaultStyle?: boolean; // 기본 메뉴 항목 스타일 사용
  className?: string;
};
```

예시:

```js title="sidebars.js"
export default {
  myHtmlSidebar: [
    // highlight-start
    {
      type: 'html',
      value: '<img src="sponsor.png" alt="스폰서" />', // 렌더링될 HTML
      defaultStyle: true, // 기본 메뉴 항목 스타일링 사용
    },
    // highlight-end
  ],
};
```

:::tip

메뉴 항목은 이미 `<li>` 태그로 감싸져 있으므로, 사용자 정의 항목이 제목과 같이 간단한 경우 값으로 문자열을 제공하고 `className` 속성을 사용하여 스타일을 지정하세요:

```js title="sidebars.js"
export default {
  myHtmlSidebar: [
    {
      type: 'html',
      value: '핵심 개념',
      className: 'sidebar-title',
    },
  ],
};
```

:::

## Category: 계층 구조 만들기 {#sidebar-item-category}

`category` 유형을 사용하여 사이드바 항목의 계층 구조를 만듭니다.

```ts
type SidebarItemCategory = {
  type: 'category';
  label: string; // 사이드바 라벨 텍스트
  items: SidebarItem[]; // 사이드바 항목의 배열
  className?: string;
  description?: string;

  // 카테고리 옵션:
  collapsible: boolean; // 카테고리를 접을 수 있게 설정
  collapsed: boolean; // 카테고리를 초기에 접혀 있거나 기본적으로 펼쳐져 있도록 설정
  link: SidebarItemCategoryLinkDoc | SidebarItemCategoryLinkGeneratedIndex;
};
```

예시:

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: '가이드',
      collapsible: true,
      collapsed: false,
      items: [
        'creating-pages',
        {
          type: 'category',
          label: '문서',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

:::tip

사용자 정의가 필요하지 않을 때는 [**축약 구문**](#category-shorthand)을 사용하세요:

```js title="sidebars.js"
export default {
  docs: {
    가이드: [
      'creating-pages',
      {
        문서: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
      },
    ],
  },
};
```

:::

### 카테고리 링크 {#category-link}

카테고리 링크를 사용하면 카테고리를 클릭하여 다른 페이지로 이동할 수 있습니다.

:::tip

카테고리 링크를 사용하여 문서 카테고리를 소개하세요.

자동 생성된 카테고리는 [`_category_.yml`](./autogenerated.mdx#category-item-metadata) 파일을 사용하여 링크를 선언할 수 있습니다.

:::

#### 생성된 인덱스 페이지 {#generated-index-page}

이 카테고리의 모든 직접적인 자식을 표시하는 인덱스 페이지를 자동으로 생성할 수 있습니다. `slug`를 사용하여 생성된 페이지의 경로를 사용자 정의할 수 있으며, 기본값은 `/category/[categoryName]`입니다.

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: '가이드',
      // highlight-start
      link: {
        type: 'generated-index',
        title: 'Docusaurus 가이드',
        description: 'Docusaurus의 가장 중요한 개념에 대해 알아보세요!',
        slug: '/category/docusaurus-guides',
        keywords: ['가이드'],
        image: '/img/docusaurus.png',
      },
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

[Docusaurus 가이드 페이지](/docs/category/guides)에서 실제로 작동하는 모습을 확인하세요.

:::tip

`generated-index` 링크를 사용하여 빠르게 소개 문서를 얻을 수 있습니다.

:::

#### 문서 링크 {#category-doc-link}

카테고리는 기존 문서로 링크할 수 있습니다.

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: '가이드',
      // highlight-start
      link: {type: 'doc', id: 'introduction'},
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

[i18n 소개 페이지](../../../i18n/i18n-introduction.mdx)에서 실제로 작동하는 모습을 확인하세요.

#### 문서 페이지에 생성된 인덱스 포함하기 {#embedding-generated-index-in-doc-page}

`DocCardList` 컴포넌트를 사용하여 일반 문서 페이지에 생성된 카드 목록을 포함할 수도 있습니다. 이는 현재 문서의 부모 카테고리의 모든 사이드바 항목을 표시합니다.

```md title="docs/sidebar/index.md"
import DocCardList from '@theme/DocCardList';

<DocCardList />
```

```mdx-code-block
<BrowserWindow>

import DocCardList from '@theme/DocCardList';

<DocCardList />

</BrowserWindow>
```

### 접을 수 있는 카테고리 {#collapsible-categories}

카테고리를 확장/축소하는 옵션을 지원합니다. 카테고리는 기본적으로 접을 수 있지만, `collapsible: false`를 사용하여 접기를 비활성화할 수 있습니다.

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: '가이드',
      items: [
        'creating-pages',
        {
          type: 'category',
          // highlight-next-line
          collapsible: false,
          label: '문서',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

모든 카테고리를 기본적으로 접을 수 없게 만들려면 `plugin-content-docs`의 `sidebarCollapsible` 옵션을 `false`로 설정하세요:

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsible: false,
        },
      },
    ],
  ],
};
```

:::note

`sidebars.js`의 옵션이 플러그인 구성보다 우선순위가 높으므로, `sidebarCollapsible`이 전역적으로 `false`로 설정되어 있어도 특정 카테고리를 접을 수 있게 만들 수 있습니다.

:::

### 기본적으로 확장된 카테고리 {#expanded-categories-by-default}

접을 수 있는 카테고리는 기본적으로 접혀 있습니다. 첫 번째 렌더링 시 확장하려면 `collapsed`를 `false`로 설정할 수 있습니다:

```js title="sidebars.js"
export default {
  docs: {
    Guides: [
      'creating-pages',
      {
        type: 'category',
        label: 'Docs',
        // highlight-next-line
        collapsed: false,
        items: ['markdown-features', 'sidebar', 'versioning'],
      },
    ],
  },
};
```

`collapsible`과 마찬가지로 전역 구성 `options.sidebarCollapsed`를 `false`로 설정할 수도 있습니다. `sidebars.js`의 개별 `collapsed` 옵션은 여전히 이 구성보다 우선순위가 높습니다.

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsed: false,
        },
      },
    ],
  ],
};
```

:::warning

카테고리가 `collapsed: true`이지만 `collapsible: false`인 경우(`sidebars.js`를 통해 또는 플러그인 구성을 통해), 후자가 우선순위를 가지며 카테고리는 여전히 확장된 상태로 렌더링됩니다.

:::

## 축약형 사용하기 {#using-shorthands}

사용자 정의가 많이 필요하지 않은 일반적인 사이드바 항목을 **축약 구문**을 사용하여 더 간결하게 표현할 수 있습니다. 여기에는 두 가지 부분이 있습니다: [**문서 축약형**](#doc-shorthand)과 [**카테고리 축약형**](#category-shorthand).

### 문서 축약형 {#doc-shorthand}

`doc` 유형의 항목은 단순히 ID를 나타내는 문자열일 수 있습니다:

```mdx-code-block
<Tabs>
<TabItem value="Longhand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      type: 'doc',
      id: 'myDoc',
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Shorthand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    'myDoc',
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

따라서 위의 예제를 다음과 같이 단순화할 수 있습니다:

```js title="sidebars.js"
export default {
  mySidebar: [
    {
      type: 'category',
      label: '시작하기',
      items: [
        // highlight-next-line
        'doc1',
      ],
    },
    {
      type: 'category',
      label: 'Docusaurus',
      items: [
        // highlight-start
        'doc2',
        'doc3',
        // highlight-end
      ],
    },
    {
      type: 'link',
      label: '더 알아보기',
      href: 'https://example.com',
    },
  ],
};
```

### 카테고리 축약형 {#category-shorthand}

카테고리 항목은 키가 라벨이고 값이 하위 항목 배열인 객체로 표현할 수 있습니다.

```mdx-code-block
<Tabs>
<TabItem value="Longhand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      type: 'category',
      label: '시작하기',
      items: ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Shorthand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      '시작하기': ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

이를 통해 예제를 다음과 같이 단순화할 수 있습니다:

```js title="sidebars.js"
export default {
  mySidebar: [
    // highlight-start
    {
      '시작하기': ['doc1'],
    },
    {
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: '더 알아보기',
      href: 'https://example.com',
    },
  ],
};
```

이 변환 후 각 축약형 객체는 정확히 하나의 항목을 포함합니다. 이제 아래의 더욱 단순화된 예제를 고려해보세요:

```js title="sidebars.js"
export default {
  mySidebar: [
    // highlight-start
    {
      '시작하기': ['doc1'],
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: '더 알아보기',
      href: 'https://example.com',
    },
  ],
};
```

연속된 두 카테고리 축약형이 하나의 객체로 압축되어 있음을 주목하세요. 이 구문은 **사이드바 슬라이스**를 생성합니다: 이 객체를 하나의 큰 항목으로 보지 말고, 각 항목이 별도의 항목이 되어 나머지 항목(이 경우 "더 알아보기" 링크)과 함께 최종 사이드바 레벨을 형성하는 것으로 보아야 합니다. 사이드바 슬라이스는 [자동 생성 사이드바](autogenerated.mdx)를 논의할 때도 중요합니다.

하나의 카테고리 축약형으로 축소된 항목 배열이 있는 경우, 그 둘러싸는 배열도 생략할 수 있습니다.

```mdx-code-block
<Tabs>
<TabItem value="Before">
```

```js title="sidebars.js"
export default {
  sidebar: [
    {
      '시작하기': ['doc1'],
      Docusaurus: [
        {
          '기본 가이드': ['doc2', 'doc3'],
          '고급 가이드': ['doc4', 'doc5'],
        },
      ],
    },
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="After">
```

```js title="sidebars.js"
export default {
  sidebar: {
    '시작하기': ['doc1'],
    Docusaurus: {
      '기본 가이드': ['doc2', 'doc3'],
      '고급 가이드': ['doc4', 'doc5'],
    },
  },
};
```

```mdx-code-block
</TabItem>
</Tabs>
```
